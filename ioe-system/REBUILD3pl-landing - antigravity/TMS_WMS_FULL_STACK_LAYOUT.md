# TMS & WMS Full-Stack Layout - Enterprise Professional

## Core Principles

1. **NO MOCK DATA** - All data from Prisma database via API endpoints
2. **NO VIBE CODING** - Professional, clean, enterprise-grade UI matching ERPNext
3. **FULL STACK** - Complete backend APIs + frontend integration
4. **ERPNext-Style Navigation** - Workspace sidebar with hierarchical structure

---

## Technology Stack (Confirmed)

### Frontend
- Next.js 15.2.4 (App Router)
- React 19
- TypeScript 5.7
- Tailwind CSS 3.4
- shadcn/ui components
- React Three Fiber (3D visualizations)
- Mapbox GL (Maps)

### Backend
- Next.js API Routes
- Prisma ORM
- PostgreSQL database
- Real-time data fetching via TanStack Query

### Integration
- ERPNext webhooks
- Real API endpoints (no mocks)
- Database-driven everything

## No Mock Data Policy

### Rules

1. **ALL data must come from API endpoints**
   - Use `fetch()` or `useQuery()` from TanStack Query
   - No hardcoded arrays or objects
   - No placeholder values like "Sample Data"

2. **ALL API endpoints must query Prisma database**
   - Use `prisma.*.findMany()`, `prisma.*.create()`, etc.
   - No mock responses or static JSON

3. **Loading States Required**
   - Show skeleton loaders while fetching
   - Show spinners for async operations
   - Handle empty states gracefully

4. **Error Handling Required**
   - Try-catch blocks around all API calls
   - User-friendly error messages
   - Retry mechanisms where appropriate

5. **Type Safety Required**
   - TypeScript types for all API responses
   - Zod schemas for validation
   - No `any` types
